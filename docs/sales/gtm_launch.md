# GTM-план запуска Aigenis Bonds (без бюджета)

> Цель: запустить продажи без денежных вложений. Весь коммерческий каркас
> (рефералка, affiliate, виджет, demo-режим, биллинг, one-pager/teaser) уже
> реализован в коде — задача этого документа свести его в работающий
> go-to-market и устранить единственный критический риск (источник данных).

## 0. Текущее состояние (что готово)
- Реферальная система (`referred_by`, `/refer`, реф-код на сайте) — ✅
- Affiliate-уровень партнёра (атрибуция подписки) — ✅
- Публичный виджет `/widget` + `WidgetPage.tsx` (iframe для партнёров) — ✅
- Demo-режим read-only (водяной знак) — ✅
- Напоминания о триале/подписке (email+TG, `reminders_daily`) — ✅
- Docs продажи: `one_pager.md`, `teaser.md` — ✅
- Fallback-источник MOEX (теперь RUB + USD/EUR евробонды) — ✅ (расширено)

## 1. Три трека продажи (приоритет)
1. **Продажа продукта компании (M&A / white-label)** — самый реалистичный исход.
   Продукт имеет production-базу (102 теста, CI, Docker, Prometheus, fail-closed
   auth) и B2B-упаковку (Partner API + webhooks + affiliate + demo). Целевые
   buyer: брокеры СНГ, агрегаторы котировок, финтех-приложения, EdTech.
2. **B2B distribution через партнёров (affiliate $0)** — дать 5–10 блогерам/
   каналам по инвестициям партнёрские ключи + виджет; они ведут трафик бесплатно
   за % с подписок.
3. **B2C органика** — контент-серия «разбор облигации по Score + Desk», публичный
   виджет на внешних площадках, Telegram-канал.

## 2. План запуска (первые 30 дней, $0)

### Неделя 1 — Инфраструктура витрины
- Поднять публичный инстанс через Cloudflare Tunnel (`DEPLOYMENT.md`, профиль
  `tunnel`): `CLOUDFLARED_TUNNEL_TOKEN=...`, `docker compose --profile tunnel up -d cloudflared`.
- Включить `DEMO_MODE=1` на поддомене `demo.` → готовая витрина для buyer и органики.
- **MOEX как полноценный источник данных** (без платного логина):
  `DATA_SOURCE=moex` (или `both`), `MOEX_BOARDS=TQCB,TQOB`. Даёт живые RUB
  корпоративы + USD/EUR евробонды. Ручной прогон: `python -m scraper moex`.

### Неделя 2 — Партнёрская сеть (affiliate)
- Найти 5–10 площадок: Telegram-каналы по облигациям, smart-lab.ru, блоги.
- Выдать партнёрские ключи (`POST /api/v1/partner/keys`) + `referral_code`.
- Дать виджет для вставки на сайт (`/widget/embed.js`).
- Настроить выплаты партнёрам (в Stars или вручную — без бюджета).

### Неделя 3 — Контент-воронка (B2C)
- Запустить Telegram-канал Aigenis Bonds: 2 поста/нед — разбор конкретной
  облигации «почему Score 82 + стресс-тест», ссылка на триал.
- Опубликовать виджет «Топ облигаций» на внешних площадках (SEO + заходы).
- Шортсы/посты на базе данных продукта (без съёмки).

### Неделя 4 — Корпоративные лиды (Трек 1)
- Разослать `one_pager.md` + `teaser.md` ЛПР брокеров/финтеха СНГ.
- Демо через `demo.`-инстанс + deep-link подписки.
- Подготовить data-room: тесты, CI, DEPLOYMENT, метрики Prometheus.

## 3. Снятие критического риска (источник данных)
- Расширен MOEX-адаптер: RUB (TQCB) + USD/EUR евробонды (TQOB), валюта берётся
  из `FACEUNIT`, лимит поднят до 1000. Продукт живёт без платного BYN-логина.
- **Честно в one_pager:** убрать «6 валют» → «RUB + USD/EUR евробонды (MOEX),
  BYN через партнёрский источник; расширение на EU/US — в roadmap».

## 4. Метрики (добавить в Prometheus, если ещё нет)
- Регистрации/нед, free→pro %, кол-во партнёров, B2B-лиды, MRR,
  трафик на `/widget`, конверсия рефералки.
- Бот уже эмитит `bot_commands_total` / `bot_errors_total` (`metrics.py`).

2. `DEMO_MODE=1` на демо-поддомене.
3. Cloudflare Tunnel → публичный доступ.
4. Разослать one_pager/teaser 10 целевым buyer.
